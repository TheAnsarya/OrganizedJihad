### Changes Made

#### `indexedDBStorage.js` — Fix #84

- Replaced all 5 `await this.ensureDB()` calls with `await this.initPromise`
- `saveAPILogs`: Simplified to delegate to existing `this.put('metadata', ...)` method
- `saveAPILogEntries`: Rewrote with proper `new Promise()` wrapping + `transaction.oncomplete`/`onerror`
- `clearOldAPILogs`: Same Promise wrapping for delete transaction
- `getAPILogs` / `getAPILogEntries`: Already delegated to `this.get()`/`this.getAll()` — only needed the `ensureDB` → `initPromise` change

#### `gameTracker.js` — Fix #85, #87

- `trackGuildWarBattle`: Added IDB `battles` store write with `battleType: 'GuildWar'`, ISO timestamp, `_isBattleDuplicate()` dedup check, matching existing arena pattern
- `trackRaidBossAttack`: Added IDB `battles` store write with `battleType: 'RaidBoss'`, `isWin: true` (raids always deal damage), ISO timestamp, dedup
- `updateChatActivitySummary`: Added `isOutgoing` parameter (default `false`). Increments `messagesSent` when `isOutgoing`, `messagesReceived` when not.
- `trackChatMessages` caller: Passes `false` (incoming messages)
- `trackOutgoingMessage` caller: Passes `true` (outgoing messages)

#### `index.js` — Fix #86

- Updated `onApiProcessed` callback to iterate `request.calls` and call `domTargeting.onApiCall(call.name)` for each API call name
- Removed stale comment about `request.name` always being undefined

#### `UpgradeTracker.js` — Fix #88

- `trackHeroStarUpgrade`: Changed `hero?.star || hero?.color || 1` → `hero?.star || 1` and `hero?.star || hero?.color || 0` → `hero?.star || 0`
- Star rank (1–6) and color/equipment rank (0–18) are entirely different progression systems

### Key Decisions

1. **saveAPILogs delegation**: Instead of rewriting the full transaction, delegated to `this.put()` which already has proper Promise wrapping — DRY and less error-prone
2. **domTargeting wiring location**: Chose to wire in `onApiProcessed` callback (index.js) rather than inside gameTracker's handler registry, because domTargeting is a DOM module instantiated in PHASE 2 and gameTracker shouldn't depend on it
3. **RaidBoss isWin**: Set to `true` always — raid boss attacks always deal damage and there's no win/loss concept; the boss eventually dies after enough guild attacks
4. **Chat isOutgoing default**: `false` — safer default since incoming messages are more common than outgoing
